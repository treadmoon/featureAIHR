# 🌍 2026 AI HR & IT Secretary (智能员工秘书) - 项目详细计划书

## 1. 项目愿景 (Project Vision)
摒弃传统“页面流”与“表单填报”的人力资源与IT工单系统，打造一个**“始终在线、意图驱动、多模态输出”**的专属 AI 员工秘书 (Employee Experience AI Secretary)。
员工不论通过语音、文字、截图还是文件提问，系统都能根据其角色（普通员工/HR/经理/IT管理员）和所处场景，实时生成最优的交互形态（语音回馈、互动式HTML卡片、图表、视频教程或一键解决动作）。实现 HR 与 IT 支持的无缝融合，达到零页面、懂意图、秒级响应的极致体验。

## 2. 核心系统架构 (Core Architecture)
- **前端与全栈框架**: Next.js (App Router) + Tailwind CSS + Framer Motion
- **AI 代理中枢 (Agent Hub)**: Vercel AI SDK / LangGraph (构建多 Agent 协同工作流)
- **后端与数据库**: Supabase (PostgreSQL 关系型数据 + Vector 向量存储支持 RAG)
- **多模态输入/输出路由器 (I/O Router)** (全面接入火山方舟大模型生态):
  - **输入**: 火山方舟·豆包语音识别 ASR (高精度语音转写), 火山方舟·豆包多模态 Vision 模型 (复杂图像/截图深入分析)
  - **输出**: v0.dev (继续用于实时生成动态交互 HTML/UI), 火山方舟·豆包文生图 (快速数据图像化/可视化), 火山方舟·豆包文生视频模型 (工作流分发与演示短视频), 火山方舟·豆包语音合成 TTS (高拟真与情感语音播报)
- **业务系统集成**: 外部系统 API 挂载 (连接如企业微信, Microsoft 365, Okta, ServiceNow 等)

## 3. 高频核心场景 (Core Scenarios)
1. **IT支持与排障**: 员工语音反馈“电脑连不上VPN” + 错误截图。AI 秘书启动 TTS 语音引导，并推送带有“自动重置”按钮的动态 HTML 卡片，附带30秒排障演示短视频。
2. **休假与考勤查询**: 员工询问“我还有多少年假”。AI 秘书即时语音回复，并生成包含饼图的 HTML 余量卡片及一键申请动作。
3. **薪酬异常解答**: 员工上传工资条截图并询问本月扣款原因。AI 识别截图，圈出重难点扣款项，推送交互式解释图表，并提供“生成调薪申请模板”的功能。
4. **全生命周期流程自动化**: 例如新员工入职流程由 AI 真人出镜短视频引导，外加上 HTML 互动打卡清单；员工离职时，AI 自主生成跨部门的 IT 权限回收指令。

## 4. 阶段性研发计划 (Phased Implementation Plan)

### 第一阶段：Spec重塑与意图场景设计 (Week 1)
- **目标**: 将传统的“功能清单”转化为“意图场景与多模态输出”列表。
- **关键里程碑**: 
  - [ ] 梳理 Top 10 的 HR+IT 高频融合场景清单。
  - [ ] 确定每个场景的默认输入方式与最佳输出模态搭配（文字/语音/交互卡片/视频）。
  - [ ] 明确需整合的外部系统接口（人事基础数据源、IT权限控制平台）。

### 第二阶段：Agent Hub 骨架与基座搭建 (Week 2)
- **目标**: 完成核心系统底座及多模态路由搭建，验证架构可行性。
- **关键里程碑**:
  - [ ] 搭建 Next.js 工程，配置环境，引入全局架构。
  - [ ] 集成 Supabase（部署核心数据库与向量存储基座）。
  - [ ] 构建去页面化的单一核心 Chat/Voice 交互入口。
  - [ ] 跑通首个测试场景流水线（例如：“查询年假”文本输入 -> 路由分发 -> 返回带图表的卡片）。

### 第三阶段：多模态 Agent 深度开发与意图拓展 (Week 3 - Week 5)
- **目标**: 针对不同角色和意图深挖各项 AI 能力。
- **关键里程碑**:
  - [ ] **Week 3 (认证与RAG)**: 实现员工、经理、HR/IT 多角色 Auth 及上下文感知；导入海量企业政策文档至向量库构建精确 RAG。
  - [ ] **Week 4 (动作执行构建)**: 接入写操作与工具调用（Tool Calling），使 AI 可以帮打卡、发审批、重置密码。
  - [ ] **Week 5 (复合型意图处理)**: 测试并完善跨领域复杂意图，如“帮我请假并帮我申请借用一台测试机”。

### 第四阶段：边界测试、集成与异常处理 (Week 6)
- **目标**: 确保系统面对复杂模糊输入时的健壮性与输出合规性。
- **关键里程碑**:
  - [ ] 测试语音嘈杂和截图模糊时的 AI 容错与 Fallback 处理机制。
  - [ ] 进行严格的权限隔离测试（如越权查询薪资的强制熔断拦截）。
  - [ ] 引入 Human-in-the-loop：当 Agent 无法确信时无缝流转至人工坐席，并保留完整审计日志。

### 第五阶段：部署上线与数据回放迭代 (Week 7)
- **目标**: 上线 MVP 并量化价值收益。
- **关键里程碑**:
  - [ ] 全栈项目发布至 Vercel 并封装至公司内聊平台（如企微/钉钉/飞书集成）。
  - [ ] 部署系统数据分析看板（关注用户查询量、意图自动化拦截率、人工升级比率等指标）。
  - [ ] 引入种子用户试运行，根据真实输入日志进行 Agent Prompt 的微调和 RAG 补充优化。

## 5. 项目收益预期 (Expected ROI)
- **研发层降本增效**: 无需编写维护数十个复杂的 CRUD 前端页面与表单，依托 UI 实生成与自然语言驱动，节省前端界面工作量超 60%。
- **用户满意度飙升**: 彻底终结新老员工“找不到入口”、“看不懂系统”的抱怨，“类人”秘书服务实现最高级别的员工自主解决率。
- **企业生态延展性**: 统一入口模型一旦成熟，不仅可服务于人力与IT，未来架构可平滑复用至财务（如差旅报销助手）、法务或销售支持系统，成为全企业极简生态的唯一大门。
