# 🚀 2026 AI HR & IT Secretary: Top 10 核心意图场景清单 (Intent Scenarios Spec)

基于“重塑 Spec”的理念，我们不再设计单独的“页面”，而是定义**10大高频场景**。针对每个场景，我们明确了用户的输入意图、多模态输出策略以及需要对接的底层系统 API。

---

## 📌 意图场景库 (Top 10 Intent Scenarios)

### 1. IT网络与设备排障 (IT Network & Device Troubleshooting)
*   **用户意图（输入）**: “我的电脑连不上公司 VPN 了” + [错误弹窗截图]（语音/图文混合）
*   **多模态输出策略**:
    *   **TTS语音**: 语音安抚并引导步骤（基于火山方舟·语音合成）。
    *   **图像解释**: 使用火山方舟 Vision 解析截图，圈出具体的错误码/网络设置项。
    *   **HTML卡片**: 推送一张包含“自动修复脚本下载”或“一键重置网卡”按钮的交互卡片（v0 生成）。
    *   **短视频**: 30秒排障步骤演示（豆包文生视频）。
*   **外部系统依赖**: IT 端侧网络配置接口 / Jamf 设备管理 API。

### 2. 休假与考勤资产查询 (Leave & Attendance Inquiry)
*   **用户意图（输入）**: “我这周调休假还剩几天？”或“我上个月考勤有异常吗？”（语音/文本）
*   **多模态输出策略**:
    *   **TTS语音**: 迅速反馈最终结论（“你本周剩余2天调休，上个月无迟到记录”）。
    *   **HTML卡片**: 展示动态甜甜圈饼图（剩余/已用份额），以及可点击的“一键发起休假审批”的联动动作。
*   **外部系统依赖**: 考勤打卡机系统 API / HRMS (例如 Workday/企微打卡) 接口。

### 3. 薪酬与绩效异常解答 (Payroll & Performance Inquiry)
*   **用户意图（输入）**: “这个月发下来的工资和我预期不一样” + [工资单流水截图]（文本/图片）
*   **多模态输出策略**:
    *   **智能图像解读**: 火山方舟 Vision 定位截图中的扣税或社保扣除明细。
    *   **HTML卡片**: 展示按类目清晰展开的交互式薪金构成梳图，并强调变动（比如五险一金基数调整）。
    *   **附加引导**: 弹出一个“提交薪资申诉”或“调薪参考意见书”的表单卡片。
*   **外部系统依赖**: 财务薪酬系统 (Payroll System) 安全查询通道 / 职级薪资带数据集。

### 4. 账号密码解锁与权限申请 (Account Unlock & Access Request)
*   **用户意图（输入）**: “帮我重置OA系统的密码” 或 “我想申请访问财务数据报表的权限”（语音/文本）
*   **多模态输出策略**:
    *   **TTS与安全交互**: 语音播报需要进行的身份认证（比如推送动态验证码）。
    *   **执行与HTML通知**: 自动执行重置后，生成显示新临时密码（阅读即焚）或权限审批跟踪的交互状态流卡片。
*   **外部系统依赖**: Okta / Azure AD / 公司自建 IAM 中台。

### 5. 新员工全生命周期入职 (Onboarding Lifecycle)
*   **用户意图（输入）**: 触发事件（入职第一天）或用户询问：“我入职需要准备什么？”
*   **多模态输出策略**:
    *   **短视频引导**: 真人/数字人出镜的入职指引视频（基于火山方舟多模态生成）。
    *   **交互式 Checklist (HTML)**: 个性化打卡清单（领取电脑、配置邮箱、签订文件），点击后触发相关子流程。
    *   **动态图像**: 公司布局 3D 导航图或组织架构树图（火山方舟文生图/HTML Canvas）。
*   **外部系统依赖**: 电子签系统接口 / 资产管理领用 API / 组织架构树源数据。

### 6. 公司政策与福利指引查询 (Policy & Benefits RAG Request)
*   **用户意图（输入）**: “关于陪产假，公司26年最新政策是什么？”、“我们有配眼镜的报销福利吗？”（语音/文字）
*   **多模态输出策略**:
    *   **精准RAG与语音摘要**: 抓取核心政策段落，利用大模型精简为3句话并通过语音播报。
    *   **交互卡片/图表**: 展示休假天数、报销额度的进度条或表格，并附带可一键点击的原文 PDF 定位链接。
*   **外部系统依赖**: 基于 Supabase Vector 的企业文档与 IT/HR 知识库 (Knowledge Base RAG)。

### 7. 审批与待办事项集中处理 (Approval & To-Do Aggregation)
*   **用户意图（输入）**: 经理/主管级询问：“今天我需要批哪些东西？”
*   **多模态输出策略**:
    *   **归纳图表 (HTML)**: 通过数据图表展示审批类型分布（例如：3个请假，2个权限开通，1个超发调薪）。
    *   **一站式操作台**: 提供高亮优先级、批量审批按钮（“全部同意并驳回超额报销”），极大简化页面跳转。
*   **外部系统依赖**: 跨系统的审批流中台 (BPM API / 飞书审批流订阅)。

### 8. 差旅与费用报销全流程 (Travel & Expense Reimbursement)
*   **用户意图（输入）**: 上传电子发票或纸质发票侧拍图：“帮我报销这趟去上海的出差”
*   **多模态输出策略**:
    *   **多模态图像识别**: 火山方舟 Vision 自动抽取出差地、金额、发票抬头及税号。
    *   **确认卡片 (HTML)**: 动态生成的报销明细表单供用户二次检查和拖拽修改，配有违规检查提示（例如超出住宿上限）。
*   **外部系统依赖**: 财务报销审批系统 / 发票平台验真 API / OCR 专用接口（视情况合并至 Vision 模型）。

### 9. 招聘流转与面试安排 (Recruitment & Interview Scheduling)
*   **用户意图（输入）**: HR级询问：“帮我筛选一下投前端工程师岗位的最近10份简历，并约通过的人明天面试”
*   **多模态输出策略**:
    *   **候选人多维度能力雷达图**: (火山方舟图表生成或定制 HTML Canvas)。
    *   **日历排期交互卡**: 直观展示招聘者与候选人时间交集，并拖拽设置会议室。
*   **外部系统依赖**: ATS 招聘追踪系统 / Microsoft 365 日历 或 Google Calendar API。

### 10. 个性化应用安装与系统配置 (Software Install & App Setup)
*   **用户意图（输入）**: 员工：“帮我在我的测试机上装一个最新版的 VSCode 和 Node.js 环境”
*   **多模态输出策略**:
    *   **执行授权与脚本下发**: 在取得验证后，卡片展示后台静默安装脚本执行的进度条 (Progress UI)。
    *   **视频/动图向导**: 提供后续配置文件的样例动图，便于开发者员工自我检查。
*   **外部系统依赖**: 终端管理统平台 / CI/CD 构建工具 (例如 GitLab CI / Ansible API 触发)。

## 🛠 关键外部接口梳理 (External API Integration Map)

根据上述意图场景，我们第一期需要 Mock/或真实对接的基础接口核心分为两类：

### HR核心能力接口群 (HR-Core)
1. **身份与组织架构 API (Identity & Org API)**：获取用户所在部门、汇报线、角色权限（决定 AI 回答尺度）。
2. **休假与考勤 API (Leave & Time API)**：查询余额、提交休假申请 webhook。
3. **政策/文档向量库 (RAG Vector Store)**：HR 手册（PDF/Word）的高效切片提取，现建议由 Supabase Vector 承载。

### IT与管理核心接口群 (IT-Core)
1. **身份认证与访问控制 API (IAM / SSO API)**：重置密码、权限组变更拦截执行。
2. **IT 资产与终端管理 API (MDM/ITSM API)**：设备网络诊断状态获取、下发安装包、获取 Ticket/工单 流转状态。
3. **日程与会议 API (Calendar API)**：实现面试预约、跨时区会议对齐功能。
